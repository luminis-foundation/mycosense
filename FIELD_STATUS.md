# MycoSense Field Deployment Status

**Last updated:** 2026-06

## Current Status: Lab / Bench Phase

MycoSense is under active development. Hardware components exist and have been tested on the bench, but a complete, continuous field deployment has not yet been validated.

| Component | Status |
|---|---|
| React dashboard (mock mode) | Stable — deployed to Vercel |
| React dashboard (live hardware mode) | Working on bench |
| Pi server (FastAPI + SQLite) | Working on bench |
| ESP32 firmware (Serial output) | Working on bench |
| ESP32 firmware (MQTT over WiFi) | Working on bench |
| Full end-to-end (ESP32 → Pi → Dashboard) | Bench-validated, not field-deployed |
| Solar power + continuous outdoor operation | Not yet validated |
| GPS surveying of node positions | Not yet completed — coordinates set to null |
| NTP synchronization | Implemented; requires WiFi at startup |
| MQTT authentication | Implemented; requires Mosquitto password file on Pi |

## What the Demo Shows

The public Vercel deployment at `mycosense.vercel.app` runs entirely in **mock mode** — all data is procedurally generated by `mockSensorData.js`. No real sensor readings are shown. This is intentional and clearly indicated by the "Mock" status indicator in the dashboard.

## What Is Not Yet Ready

- Continuous unattended outdoor operation (solar + battery runtime not validated)
- Weatherproofing of enclosures
- GPS coordinates for electrode placement
- Production MQTT TLS configuration
- Automated firmware OTA updates
- Data retention policy for long-term logging

## Research Anchor

The associated preprint ([DOI 10.5281/zenodo.20143391](https://doi.org/10.5281/zenodo.20143391)) describes the biosensor methodology and signal processing approach. The software infrastructure described here is the instrumentation implementation of that research program.

Provenance hashes generated by the dashboard anchor session datasets to the Luminis Foundation via SHA-256. On-chain posting (Polygon / Gnosis Safe) is planned for Phase 3.
